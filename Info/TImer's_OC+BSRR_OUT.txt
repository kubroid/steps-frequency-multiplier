Как и в OC_DMA, входной буфер не нужен. 
Мы лишь считаем кол-во входных шагов для каждой оси.
А при выводе отнимаем от этого числа, пока оно не станет = 0.
Для отлова входных шагов и подсчета частот используем EXTI.

Выходной таймер всегда должен работать на максимальной из входных частот.
Период счётчика всегда постоянен.
А предделитель меняется каждую миллисекунду (sys tick).
Для вывода используем метод таймера Output Compare (Time base)
вкупе с атомарной записью в порт (BSRR).

----------- ввопрос ---------
Вопрос только в том, как именно использовать событие OC 
выходного таймера для вывода всех осей.

----------- ответ ----------
Как и в OC_DMA, используем заранее созданный массив значений для OC,
а далее используем DMA для переноса значений массива в регистр сравнения.

Входной период каждой оси вычисляется каждую миллисекунду
на основе засечки по времени, сделанной в прерывании EXTI.
Самый малый период становится предделителем для выходного таймера.

Для каждой оси будет обратный счетчик (делитель).
Если он = 0, то в прерывании ОС дёргается выходной пин оси,
и значение счетчика сбрасывается на стандартное, равное делителю.
    ПОДРОБНЕЕ..
    Стандартное значение обратного счетчика оси расчитывается 
    каждую миллисекунду на основе периодов всех осей.
    Допустим, период оси Х = 60 мкс, и он самый малый из всех. 
    А период оси Y = 600 мкс. Значит, стандартное значение обратного
    счетчика для оси Х = 1 (60/60), а для оси Y = 10 (600/60).
    В каждом прерывании ОС обратный счетчик каждой оси уменьшается на 1.
    И как только он станет = 0, дергается выходной пин указанной оси.

Значение обратного счетчика каждой оси может уменьшаться или увеличиваться
каждую миллисекунду, в зависимости от текущего предделителя выходного
таймера.
    ПОДРОБНЕЕ..
    Допустим, период оси Х = 60 мкс, и он самый малый из всех. 
    А период оси Y = 600 мкс. Значит, стандартное значение обратного
    счетчика для оси Х = 1 (60/60), а для оси Y = 10 (600/60).
    Текущее значение обратного счетчика оси Х сейчас равно 1,
    и оно не меняется, т.к. до этого стандартный . 
    А для оси У, у которой обратный счетчик = 30,
    значение должно поменяться.
